Thank you @Drew Breunig — your recent YouTube video directly inspired what I'm releasing today.

Your Spec-Driven Development Triangle and Plumb tool asked the right question: how do we keep specs, tests, and code in sync? But I wanted to add one more question. Why? Why does this code exist? What's its PURPOSE? Trillions of lines of code every day, and yet, why do I think that's all insufficient?

That question became a framework. And the framework became five open-source tools on GitHub.

---

Introducing Hedonics Driven Development (HDD) — classify software, policy, and life decisions by the human purposes (hedonics) they serve.

Five packages. All open source on GitHub: github.com/altpath-ai/highnoon

Each works as a CLI, an MCP server (add to Claude/GPT/Cursor), and a Python library.

pip install hedonics      # The shared taxonomy — what Hedonics are we optimizing?
pip install altpath       # Personal life assessment — what Hedonics are important to me?
pip install highnoon      # Software purpose assessment — tag software by Hedonic Teleology classifications
pip install mainstreet    # Public policy assessment — how does this policy improve individuals' hedonic experience?
pip install frontpage     # Hedonic content discovery — personal newsfeed based on my personal hedonic calculus

---

We classify software by what it's BUILT WITH — Python, React, microservices. But what if we classified it by what it's BUILT FOR?

Every piece of software serves human purposes (ENDS) and costs human resources (MEANS). Many ends are locked behind means costs. The framework identifies both — and computes how to trade surplus resources for deficit resources to unlock what matters.

That's Teleological Hedonic Optimization. Not just assessment — allocation.
