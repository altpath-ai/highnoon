Thank you @Drew Breunig — your recent YouTube video directly inspired what I'm releasing today.
Your Spec-Driven Development Triangle and Plumb tool (SDD) is a huge leap forward in the world we work and live in today.

Introducing Hedonics Driven Development (HDD). It starts where SDD starts — specs, tests, code — and adds one word to each: Hedonic Specs. Hedonic Tests. Hedonic Code. And then one node SDD never had: Hedonic Outcomes. Because the code isn't done when the tests pass. It's done when someone's life improves.

GitHub: github.com/altpath-ai/highnoon

Each code set works as a CLI, an MCP server (add to Claude/GPT/Cursor), and a Python library.

pip install altpath       # What Hedonics are important to ME?
pip install frontpage     # My personal newsfeed based on my hedonic calculus
pip install hedonics      # What Hedonics are we optimizing?
pip install highnoon      # Tag software for SEO by Hedonic Teleology classifications
pip install mainstreet    # How does this policy improve individuals' hedonic experience?

The framework measures your life across 10 hedonic domains — Nourishment, Shelter, Health, Care, Mobility, Growth, Connection, Recreation, Expression, Meaning — and 9 fungible cost categories that block access to those domains. (see: AltPath.ai)

Then it computes how to trade what you HAVE (Time, Money, Skills, Resources) for what you NEED (housing, food, healthcare, transportation, etc). That's the fungibility calculus. You have time but little money? Trade time for money. You have knowledge but no skills? Knowledge leads to skills through application. Every exchange unlocks a door to something that actually matters to you.

You have a great life. You could have a better one. Your best life isn't hidden. It's locked. Behind costs you can see but haven't named. HDD names them, measures them, and finds the keys.

All five packages are live on PyPI and GitHub. Open for stars, issues, PRs, and discussion.
